# API Организационной структуры

REST API на базе Django для управления организационными структурами, включая отделы и сотрудников. Поддерживает иерархические связи, обнаружение циклов и гибкие режимы удаления.

## Особенности
- **Иерархия отделов**: Неограниченная глубина (получение через API ограничено глубиной 5 уровней).
- **Обнаружение циклов**: Предотвращает перемещение отдела в собственное поддерево.
- **Иерархическая уникальность**: Названия отделов должны быть уникальными внутри одного родительского отдела.
- **Настраиваемые режимы удаления**:
  - `cascade`: Полное удаление данных подразделения, отделов, сотрудников.
  - `reassign`: Удаление подразделения, отделов и перенос сотрудников в другое подразделение.
- **OpenAPI документация**: Доступна через Swagger и ReDoc.

## Начало работы

### Требования:
- Docker
- Docker Compose

### Запуск проекта
1. Клонировать репозиторий
   ```bash
   git clone https://github.com/DmitryHook/hhtest
   cd hhtest
   ```
2. Запустите приложение с помощью Docker Compose:
   ```bash
   docker-compose up --build -d
   ```
3. API будет доступно по адресу `http://localhost:8000`.

### API документация
После запуска сервера вы можете изучить API по адресам:
- **Swagger**: `http://localhost:8000/departments/schema/docs`
- **ReDoc**: `http://localhost:8000/departments/schema/redoc`

### Запуск тестов
Чтобы запустить набор тестов, используйте команду:
```bash
docker-compose exec web pytest
```

## API Endpoints

### Departments
- `POST /departments/`: Создает новое подразделение, можно указать родительский отдел через parent_id
- `GET /departments/{id}/`: Возвращает детальную информацию о конкретном подразделении, включая его сотрудников и иерархическую структуру вложенных отделов
- `PATCH /departments/{id}/`: Редактирование данных подразделения, можно сменить название через name или родительский отдел через parent_id
- `DELETE /departments/{id}/`: Удаление подразделения с выбором стратегии: cascade - полное удаление данных подразделения, отделов, сотрудников; reassign - удаление подразделения, отделов и перенос сотрудников в другой отдел

### Employees
- `POST /departments/{id}/employees/`: Создает сотрудника в подразделении, можно указать дату найма через hired_at

## Структура проекта
- `config/`: Конфигурация проекта (настройки, глобальные URL).
- `departments/`: Основная логика приложения (модели, сериализаторы, представления, миграции).
- `tests/`: Комплексный набор тестов.
